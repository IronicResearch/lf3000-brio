Detailed Description

Module Public Interface (MPI) class for Audio module.

The audio system is composed of three basic parts: players, channels, and a mixer. When a user calls StartAudio, this creates a player, connects it to a channel, launches the player's audio stream. As the audio is rendered from the player, it is mixed with other channels by the mixer, and finally output. When the player's stream ends, an event is posted to the listener if one has been supplied. Subsequently, the player is destroyed and the channel that it was using becomes free for another player.

The audio mixer always outputs single stereo stream at 32KHz sample rate. Supported player sample rates are 32KHz, 16KHz, or 8KHz, stereo or mono. The current AudioMPI implementation supports the following codec and file types:

  • Ogg Vorbis (audio files with .ogg extensions)
  • WAV ADPCM (audio files with .wav extensions)
  • Uncompressed PCM (audio files with .wav, .avi, or raw .brio headers)

The maximum number of concurrent player streams and mixer channels supported are 5 Ogg Vorbis plus 16 PCM/ADPCM. Playback performance subject to CPU loading.

Players are represented internally by a tAudioID. The tAudioID is returned when the player is created with a call to StartAudio. Whenever the user wishes to operate on a particular player's stream, he will have to provide its tAudioID. The tAudioID ceases to be valid after a kAudioCompletedEvent or a call to StopAudio. If a listener is provided for these events, that listener should assume that the tAudioID is not valid in its Notify function.

Reasons that audio players may not get created and return kNoAudioID are:

  • (1) The audio file path is not valid.
  • (2) Too many players of given codec type are already playing.
  • (3) Too many players of higher priority are already playing.

Calling the various MPI functions with an invalid tAudioID will either result in no action or an error.

The user can start a player with a given priority. If no channel is available, and a lower priority player is playing, that player will be terminated in order to free up a channel. (See "About Priority Policies")

MIDI functions are no longer implemented.

AudioEffectsProcessor features are not implemented.

Play an audio resource

Create a player and add its output stream to the system. The stream will stop under one of the following circumstances:

  1. The end of the player's audio resource is reached and the kAudioOptionsLooped flag is not specified. This is a kAudioCompletedEvent.
  2. The kAudioOptionsLooped option is passed and the stream has been repeated as many times as indicated by the payload parameter. This is also a kAudioCompletedEvent.
  3. The user calls StopAudio with the player's tAudioID.
  4. The priority policy terminates the player's stream because there are no channels available and somebody called StartAudio for a player of higher priority.
  5. The AudioMPI destructor is called. Any streams that are playing at this time will be terminated.

Numerous parameters affect the behavior of the player. Specifically:

Parameters
pathThis is the path to the audio resource to be played. If it begins with a / it is considered absolute. Otherwise, it is relative to the path returned by GetAudioResourcePath.
volumeThe volume, between kAudioVolumeMin and kAudioVolumeMax at which the stream should be played. This volume can be inspected and adjusted later by calling GetAudioVolume and SetAudioVolume respectively.
priorityIn the event that all channels are busy AND a player of lower priority is currently playing, a lower priority player will be stopped to free a channel. Similarly, if a future call to StartAudio happens with a higher priority, then this player may be terminated. priority is an unsigned value from 0 to 255 with 255 being the highest priority.
panThe pan is a signed value between kAudioPanMin and kAudioPanMax. This pan can be inspected and adjusted later by calling GetAudioPan and SetAudioPan respectively.
pListenerThis is the listener to whom all audio events related to this player will be posted. Passing NULL means that no events will ever be posted, even if the default listener is set to a non NULL value using SetDefaultAudioEventListener.
payloadThe payload parameter is a user-supplied argument that is passed to the listener when various events occur. However, for legacy reasons, there is one overloaded use case. When the kAudioOptionsLooped flag is set, the payload parameter represents the number of times the audio should be looped. Pass kAudioRepeat_Infinite to repeat forever. In this legacy case, the payload parameter will still be passed to the listener.
flagsThe flags parameter is a collection of OR'd values that influence many of the player's behaviors. See tAudioOptionsFlags for details on which flags are available and how they influence the behavior of the player.
Returns
This function returns kNoAudioID on failure and a valid tAudioID on success.
