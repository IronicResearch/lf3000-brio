312 // Used at task creation and when changing the properties of a running task.
313 // Callers can query the Kernel for preferred task properties using
314 // CKernelMPI::GetTaskPropertiesPreferences(...) and the appropriate
315 // tTaskOperatingMode.
316 //
317 // tTaskProperties contains all the task properties values.
318 // tTaskPropertiesSet defines a complete or subset group of these properties
319 // that apply to a specific task running in a specific operating mode.
320 //
321 // 1.
322 // 2.
323 // 3.
324 // 4.
325 // 5.
326 // 6.
327 // 7.
328 //
329 // 8.
330 // The supported values of policy shall include SCHED_FIFO, SCHED_RR, and SCHED_OTHER,
331 // which are defined in the <sched.h> header. When threads executing with the scheduling
332 // policy SCHED_FIFO, SCHED_RR, or SCHED_SPORADIC are waiting on a mutex, they shall acquire
333 // the mutex in priority order when the mutex is unlocked.
334 //
335 // 9.
336 //
337 // 10.
338 // The inheriSsched attribute determines how the other scheduling attributes
339 // of the created thread shall be set:
340 //
